## Project Overview

Spring Frontend — the client for a competitive programming / placement-helper platform for NIT Kurukshetra. The backend API is described in `api_specs.json` (OpenAPI 3.0.3) and the full product spec lives in `client-specs.md`.

## Commands

- `bun install` — install dependencies
- `bun run dev` — start Vite dev server
- `bun run build` — typecheck (`tsc -b`) then production build
- `bun run preview` — preview production build

## Tech Stack

- React + TypeScript + Vite
- TanStack Router (file-based routing via `@tanstack/router-plugin/vite`)
- TanStack Query for data fetching
- shadcn/ui v3 (uses `@base-ui/react`, NOT Radix — use `render` prop instead of `asChild`)
- Tailwind CSS v4 (via `@tailwindcss/vite` plugin, no `tailwind.config` file)
- Better Auth client (`better-auth/react`)
- Monaco Editor (`@monaco-editor/react`)
- Package manager: **bun**

## Key Architecture

- **Route tree auto-generation:** `src/routeTree.gen.ts` is generated by the TanStack Router Vite plugin on dev server start. Never edit it manually.
- **Auth guard:** `src/routes/_authenticated.tsx` is a layout route whose `beforeLoad` checks the session via `/api/auth/get-session`. All routes under `src/routes/_authenticated/` inherit this guard. `login.tsx` sits outside it (public).
- **Root layout:** `src/routes/__root.tsx` provides the QueryClient via router context.
- **API client:** `src/lib/api.ts` — single `apiFetch` wrapper with `credentials: "include"` and base URL from `VITE_API_BASE_URL` env var.
- **Auth client:** `src/lib/auth.ts` — Better Auth `createAuthClient` pointing at `${VITE_API_BASE_URL}/api/auth`.
- **Path alias:** `@/` maps to `src/` (configured in both `vite.config.ts` and `tsconfig.app.json`).

## shadcn/ui v3 Notes

Components use `@base-ui/react` primitives, not Radix. Key difference: **no `asChild` prop**. To compose a component as a different element, use the `render` prop:

```tsx
// Correct (base-ui)
<Button render={<Link to="/foo" />}>Click</Button>
<DropdownMenuItem render={<Link to="/bar" />}>Item</DropdownMenuItem>

// Wrong (old Radix pattern)
<Button asChild><Link to="/foo">Click</Link></Button>
```

## API

Backend base URL is set via `VITE_API_BASE_URL` env var (see `.env.example`). Key endpoint groups (see `api_specs.json` for full details):

- Auth: Better Auth endpoints under `/api/auth/`
- Problems: `/problemset/`, `/problemset/:slug`
- Contests: `/contests/`, `/contests/:contestNumber`
- Submissions: `/submissions/`, `/submissions/:id`
- Languages: `/languages/`
- Run/Submit: POST `/run`, POST `/submit` (body: `slug`, `engineLanguageId`, `sourceCode`)
- Server time: `/time` (use this for contest status, not browser time)
